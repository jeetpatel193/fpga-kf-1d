Project Update 1 — FPGA-Accelerated 1D Kalman Filter (IMU only)

Brief description
A runnable Python golden model of a 1-D Kalman Filter for IMU sensor fusion (single-axis angle).
Model: A=1, B=Δt, C=1. Predict uses gyro rate (input u_k), Update uses accelerometer angle (measurement z_k).
We provide two scenarios (sine and step), write golden outputs (x_hat, P, K) to CSV, generate plots, and include a
scripted checker to compare future HLS outputs against the golden arrays.

What I implemented
• Data generator (golden/gen_imu.py): creates t, x_true, u_gyro (rad/s), z_accel (rad) + plots.
• 1-D KF runner (golden/kf_imu_from_csv.py): computes x_hat, P, K + plots and RMSE vs ground truth.
• Auto-checker (golden/compare_golden.py): MAE/RMSE/MAX and PASS/FAIL vs golden.

How to run
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python golden/gen_imu.py --scenario sine --plots
python golden/gen_imu.py --scenario step --plots
python golden/kf_imu_from_csv.py --scenario sine --Q 1e-4 --R 1e-2 --P0 1.0 --plots
python golden/kf_imu_from_csv.py --scenario step --Q 1e-4 --R 1e-2 --P0 1.0 --plots

verification
python golden/compare_golden.py --ref data/sine_x_hat.csv --cand data/sine_x_hat.csv --eps 1e-9 --label x_hat   # PASS
python golden/compare_golden.py --ref data/sine_P.csv     --cand data/sine_P.csv     --eps 1e-9 --label P       # PASS
# RMSE improvement (from kf_imu_from_csv.py):
# sine: measurement RMSE ≈ 0.0513 → KF RMSE ≈ 0.0114
# step: measurement RMSE ≈ 0.0513 → KF RMSE ≈ 0.0132
# Steady-state (Q=1e-4, R=1e-2): K∞ ≈ 0.0951, P∞ ≈ 9.512e-4 (matches plots/CSV tails)

Environment
Python 3.x (tested on 3.12), numpy, matplotlib (see requirements.txt). Ran on WSL/Ubuntu.
